2024 vnctf CutePath

抓包，是一个share目录

放包过程发现

这个filepath应该有问题

最开始去谷歌直接搜索题目,当然是什么都找不到了,然后去bing搜CutePath server然后就发现了Cute HTTP File Server,然后又回谷歌找了好久,在github一个项目的issues找到了他,虽然但是这样我还是没做出来,哎

是要根据重命名这个功能移动文件到share目录下,因为最开始抓包就是share目录,这个地方可以下载文件

接着回到首页,(正常的share目录是不能下载的),

然后就看到了梦寐以求的flag

VNCTF{564e406840636b3156315f6764676d}

[还有种思路：]( VNCTF2024 writeup(含cve-2024-0603漏洞分析) - 先知社区 (aliyun.com) )

修改路径为/chfs/downloaddir/../../../../flag就能够成功获取flag目录的所有内容,包括flag

总结

知识点:

Cute HTTP File Server存在几个漏洞,目录遍历、任意目录移动、任意文件移动、任意目录下载。

题目最开始给你一个文件下载漏洞,然后根据自己又发现的移动文件漏洞,下载flag
